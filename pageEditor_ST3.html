📄 Pages Implemented

Team Page

This team page was designed to introduce all group members and clearly present their roles and contributions to the website. Each member is represented through a circular image to create a clean and visually engaging layout, while hover effects were implemented using CSS to enhance interactivity. When users hover over a team member, additional details such as their role, assigned pages, and a brief description of their contribution are revealed, improving user experience without cluttering the interface. In terms of implementation, HTML was used to structure each team member as an individual component, while CSS Flexbox was applied to align the members in a responsive layout. Hover effects and transitions were created using CSS to ensure smooth animations. Accessibility considerations were also included by adding alternative text for images and enabling keyboard focus on each member block. This page demonstrates both visual design consistency and effective use of interactive elements to present team information in a user-friendly manner.

Accessibility

The Team Page can be accessed through the navigation bar located at the top of the website by selecting the “Team” link. This page allows users to view all group members, their roles, and their contributions to the project. The layout is designed to be clear and interactive, enabling users to easily explore information about each team member.

FeedBack Page

The Feedback and Program Directory page was designed to provide users with an interactive platform to explore sustainability-related programmes and share their opinions. The page includes a scrollable program section, a ratings and reviews summary, and a feedback form enhanced with JavaScript to improve user interaction. The program cards were structured using HTML and styled with CSS to create a clean and consistent layout, while hover effects were applied to improve visual engagement. The feedback form was implemented using semantic HTML elements such as fieldsets and labels to ensure proper structure and accessibility. JavaScript was used to enhance the functionality of the form by adding a live character counter for the message input, validating user inputs, and displaying real-time error messages. A success message is shown upon valid submission, providing immediate feedback to the user. These features demonstrate the separation of structure, presentation, and behaviour, as HTML, CSS, and JavaScript are used together to create an interactive and user-friendly experience. Overall, this page reflects both technical implementation and user-centred design.

Accessibility

The Feedback Page is accessible via the navigation bar by clicking on the “Feedback” option. This page provides users with the opportunity to explore available programmes, read user reviews, and submit their own feedback through an interactive form. It is designed to be user-friendly and easy to navigate.

Content page

I The Content Page was developed to present detailed information about Ocean Acidification Awareness in a structured and user-friendly format. The page was organised into multiple sections using appropriate HTML heading elements, ensuring a clear hierarchy and logical flow of information. Each section focuses on a specific aspect of the topic, such as its causes, impacts on marine life, and possible solutions, allowing users to easily understand complex environmental issues. To improve navigation within the page, internal links were implemented using anchor tags, enabling users to quickly jump between different sections. A “Go to Top” button was also added using CSS positioning to enhance usability, especially for longer content. Relevant images were included to support the text, and each image was provided with meaningful alternative text to ensure accessibility. CSS was used to style the layout, control spacing, and maintain consistency with the overall website design. This page demonstrates effective content structuring, internal navigation, and accessibility considerations while presenting informative and engaging content.

🧠 Challenges and Lessons Learned

During the development of the website, several challenges were encountered, particularly in implementing interactive features and maintaining a consistent design across all pages. One of the main difficulties was ensuring that hover effects and dynamic elements, such as the team member details and feedback form interactions, functioned correctly without affecting the overall layout. This required careful use of CSS positioning and transitions to avoid layout shifting and ensure a smooth user experience. Another challenge was integrating JavaScript functionality into the feedback form, including implementing a live character counter and validating user inputs. Initially, handling form validation and displaying error messages in the correct positions proved difficult, but this was resolved by using DOM manipulation and event listeners effectively. Additionally, ensuring accessibility and proper structure, such as using semantic HTML elements, labels, and alternative text, required attention to detail. Through these challenges, valuable lessons were learned about the importance of separating structure, presentation, and behaviour using HTML, CSS, and JavaScript. The experience also improved problem-solving skills, debugging techniques, and the ability to design user-friendly and interactive web interfaces. Overall, the process contributed to a deeper understanding of modern web development practices.

♿ Accessibility

  • Alt text: Every <img> element includes descriptive alt attributes that convey the image content and context
  • Semantic HTML: Pages use <header>, <nav>, <main>, <footer>, <section>, and <article> elements for proper document structure
  • Keyboard usability: Gallery items have tabindex="0" and keydown listeners for Enter/Space activation. Modal can be closed with Escape key
  • ARIA attributes: Navigation uses aria-label, modal uses role="dialog" and aria-modal="true", interactive elements have aria-label
  • Readable contrast: Dark text (#023047) on light backgrounds (#f0f8ff) provides a contrast ratio exceeding WCAG AA requirements
  • Skip link: Footer accessibility link jumps to #main-content
  • Responsive text: Font sizes scale with media queries for readability on all devices

🔒 JANET Compliance

This website adheres to responsible web publishing standards as required by the JANET Acceptable Use Policy:

  • Privacy: No personal data is collected, stored, or transmitted. No cookies or tracking mechanisms are used
  • Accuracy: All statistics and claims are sourced from reputable organisations (UN, UNEP, Ocean Conservancy)
  • Security: No server-side processing or database connections. All interactions are client-side only
  • Accessibility: The site follows WCAG 2.1 guidelines with semantic HTML and accessible navigation
