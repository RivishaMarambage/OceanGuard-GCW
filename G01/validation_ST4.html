📋 Validation Process

Each page was validated using the W3C Markup Validation Service. Screenshots below show evidence of successful validation.

👤 profile.html – User Dashboard Report

Links: View Page | View Documentation

Validation Notes

  • Dynamic Content: Validated that the JavaScript-driven innerHTML updates do not break the HTML5 structure.
  • Aria Compliance: Confirmed that custom progress bar containers use valid ARIA attributes for screen reader compatibility.
  • Multimedia: Validated the <video> background tag structure and fallback logic.

Reflection

The User Profile page required careful auditing of the interaction between CSS transitions and the JavaScript prompt logic. By adhering to W3C standards, I ensured that the interactive "Guardian Rank" dashboard remains stable even when users provide unexpected input during the setup process.

🗺️ sitemap.html – Interactive SVG Report

Links: View Page | View Documentation

Validation Notes

  • SVG Schema: The <svg> element was validated against XML/HTML5 namespaces to ensure the viewBox scales correctly.
  • Interactive Nodes: Confirmed that nested anchor tags (<a>) within the SVG coordinate system are structurally valid.
  • Fluidity: Verified that the removal of fixed width/height attributes in favor of percentage-based widths passed without warnings.

Reflection

Developing the sitemap was the most technical part of the individual tasks. Using the viewBox attribute was essential for responsiveness. Validation confirmed that the complex nested structure of rectangles and text nodes remains accessible and standards-compliant.
