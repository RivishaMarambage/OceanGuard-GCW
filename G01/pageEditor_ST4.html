Sitemap (sitemap.html)

Technical Description

The sitemap was built entirely from scratch using Scalable Vector Graphics (SVG) rather than a static image or standard HTML divs. This ensures resolution independence and interactivity.

  • Coordinate System: Utilised a viewBox="0 0 1200 660" to create a responsive, mathematical grid. A strict 60px vertical rhythm was used to layout the sub-page branches.
  • Interactive Nodes: Shapes (<rect>) and text were wrapped in standard <a> tags to create clickable website navigation directly inside the SVG graphic.
  • CSS Styling: Implemented a hierarchical color scheme using CSS classes (.node-parent, .node-child) to represent different levels of website depth visually. Hover transitions were added to improve user feedback.
  • Structural Layering: Lines were coded before shapes to take advantage of the SVG painter's algorithm, ensuring connecting branches rendered cleanly behind the interactive buttons.

Accessibility

Non-clickable category headers were wrapped in <g> (group) tags rather than anchor links to prevent user confusion. The container was styled with overflow-x: auto to ensure horizontal scrolling on mobile devices rather than breaking the layout.

🧠 Challenges and Lessons Learned

The biggest challenge I faced was developing the interactive SVG Sitemap. Mapping out the exact X and Y coordinates for every box and connecting line required precise mathematical planning. When I first applied thick strokes to the lines, they created gaps at the 90-degree corners. I overcame this by researching SVG properties and implementing stroke-linecap: round in CSS, which fixed the joints seamlessly.

I also learned a great deal about the importance of the CSS Box Model and Overflow properties. Initially, my large SVG map caused horizontal scrolling issues on small screens. By wrapping the SVG in a responsive container with max-width: 100% and overflow-x: auto, I successfully protected the site's layout on mobile devices.

♿ Accessibility

  • Alt text: Every <img> element includes descriptive alt attributes that convey the image content and context.
  • Semantic HTML: Pages use <header>, <nav>, <main>, <footer>, and <article> elements for proper document structure.
  • Visual Hierarchy: The sitemap uses a distinct cascading color palette (from dark navy to light pastel blues) to communicate depth and structure intuitively without relying solely on text.
  • Readable contrast: Dark text is used on light backgrounds across the profile and content pages to ensure contrast ratios exceed WCAG AA requirements.
  • Responsive text & Containers: Font sizes scale appropriately, and large diagrams (like the sitemap) are contained within scrollable divs to prevent screen-breaking on mobile assistive devices.
