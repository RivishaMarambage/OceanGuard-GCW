📄 Pages Implemented

Shared HTML Template

Every page follows a consistent template structure:

  • <header> – Contains the website logo/title and the main navigation bar with links to all pages.
  • <nav> – Semantic navigation element with an unordered list of links. Uses aria-label for accessibility.
  • <main> – Primary content area unique to each page, identified with id="main-content".
  • <footer> – Contains links to the Page Editor and Validation pages, student details, and an accessibility link.

Navigation Bar

The navigation bar is implemented as a flex-based horizontal list inside a <nav> element. Key features include:

  • Hover effects using CSS :hover pseudo-class with background colour transitions
  • Active page highlighting via a CSS .active class applied to the current page link
  • Responsive wrapping using flex-wrap for smaller screens
  • Keyboard-accessible focus indicators for tab navigation

Global CSS (style.css)

A single CSS file provides consistent styling across all pages. Techniques used:

  • CSS Custom Properties (Variables): Defined in :root for colours, shadows, spacing, and fonts
  • CSS Grid & Flexbox: Used for responsive card layouts, gallery grids, and header/footer structure
  • Media Queries: Breakpoints at 900px (tablet) and 600px (mobile) for responsive design
  • CSS Transitions & Animations: Smooth hover effects, card lifts, and modal entrance animations using @keyframes
  • Google Fonts: Inter for body text and Playfair Display for headings
  • Box-shadow layering: Multiple shadow levels (sm, md, lg) for depth hierarchy

Home Page

Technical Description

The home page uses the shared template and includes:

  • Hero section: Full-width banner with gradient background and SVG wave decoration
  • Mission Cards: Three <article> elements in a CSS Grid, each with an image, heading, paragraph, and action list
  • Gallery Tiles: Four clickable tiles linking to content pages with overlay gradient and hover scale effect
  • Embedded CSS: Six <style> rules for tile hover animation, card emphasis, list styling, image overlay, title positioning, and grid layout

HTML tags used: <section>, <article>, <h1>–<h3>, <p>, <ul>, <li>, <img>, <a>, <div>, <span>

Accessibility

Semantic structuring ensures screen readers can navigate easily. Alt texts were provided for all images, and color contrast ratios were tested to meet WCAG standards. Navigation is keyboard accessible with focus indicators.

Gallery Page

Technical Description

The gallery page includes six ocean conservation images in a responsive grid with full JavaScript interactivity:

  • Hover highlighting: mouseenter/mouseleave event listeners toggle a .highlighted CSS class
  • Click-to-expand modal: Clicking a thumbnail reads data-* attributes and populates a modal overlay
  • Close mechanisms: Close button, clicking outside the modal, and Escape key
  • User customisation: Font selector (<select>) and colour theme selector that modify modal styles via JavaScript

JavaScript techniques: querySelectorAll, addEventListener, classList.add/remove, getAttribute, DOM style manipulation

Accessibility

All gallery items have tabindex="0" and keyboard event listeners for Space and Enter. The modal is assigned role="dialog" and aria-modal="true", and can be closed using the Escape key to ensure smooth keyboard usability.

Content Page

Technical Description

The content page covers "Plastic Pollution & Ocean Protection" with:

  • Internal navigation: Anchor links to four page sections using id attributes
  • Four sections: The Problem, Effects on Marine Life, Student Actions, and Cleanup Innovations
  • Image-text layouts: Flexbox-based side-by-side content using the .content-with-image class
  • Go To Top button: CSS position: fixed button linked to a top anchor — no JavaScript used

Accessibility

High-contrast colors and semantic heading structures improve overall readability. The internal anchor links help users with assistive technologies to seamlessly jump between relevant sections.

🧠 Challenges and Lessons Learned

A primary challenge was implementing the JavaScript gallery modal without relying on external libraries. Tracking focus states and ensuring the modal was fully keyboard-accessible required extensive logic refactoring. From this, I learned the importance of native DOM APIs and ARIA roles for inclusive design.

♿ Accessibility

  • Alt text: Every <img> element includes descriptive alt attributes that convey the image content and context
  • Semantic HTML: Pages use <header>, <nav>, <main>, <footer>, <section>, and <article> elements for proper document structure
  • Keyboard usability: Gallery items have tabindex="0" and keydown listeners for Enter/Space activation. Modal can be closed with Escape key
  • ARIA attributes: Navigation uses aria-label, modal uses role="dialog" and aria-modal="true", interactive elements have aria-label
  • Readable contrast: Dark text (#023047) on light backgrounds (#f0f8ff) provides a contrast ratio exceeding WCAG AA requirements
  • Skip link: Footer accessibility link jumps to #main-content
  • Responsive text: Font sizes scale with media queries for readability on all devices

🔒 JANET Compliance

This website adheres to responsible web publishing standards as required by the JANET Acceptable Use Policy:

  • No copyright infringement: Images are generated or used with appropriate permissions. All external sources are cited in the references section

  • Privacy: No personal data is collected, stored, or transmitted. No cookies or tracking mechanisms are used
