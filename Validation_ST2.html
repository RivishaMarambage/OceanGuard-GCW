HTML & CSS Validation

Validation Notes

  • Result:content_ST2.html passed W3C validation with zero errors.
  • HTML entity encoding:The & entity in the page title ("Coral Reef Protection & Sustainability") was correctly encoded and accepted without errors.
  • Heading hierarchy:One <h1> for the page title followed by <h2> section headings — the validator confirmed this logical structure is correctly ordered.
  • Image alt text:All four content images include meaningful alt attributes describing the image content — no missing alt text errors were found.
  • Internal anchor links:All four navigation links (href="#importance", href="#threats", href="#affects", href="#actions") correctly match their corresponding section id attributes.
  • Semantic elements:Use of <main>, <section>, <nav>, <header>, and <footer> was confirmed as correctly structured.
  • meta description tag:The <meta name="description"> tag was placed correctly inside <head> and accepted without any errors.
  • Go to Top button:The <a href="#top"> anchor link and the corresponding <a id="top"> at the top of the page were both validated correctly with no errors.

Reflections

The Content Page passed validation with zero errors, which I attribute to consistently using semantic HTML elements and correctly encoding special characters throughout development. The internal navigation links were a key feature of this page, and validation confirmed that all anchor IDs matched their corresponding section targets perfectly. This process reinforced how important it is to plan the page structure and heading hierarchy carefully before writing content..
